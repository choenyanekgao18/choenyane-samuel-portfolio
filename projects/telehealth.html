Project overview

TeleHealth supports selected healthcare services online, helping patients and practitioners manage appointments, consultations, payments, prescriptions, sick notes, and records from one connected platform.

The project was built as a branded healthcare solution with a structure that can be expanded to support multiple healthcare practitioners. Client-specific branding and sensitive information have been removed this case study.

Real-world context

WhyTeleHealthmatters.

In a private healthcare context, not every patient interaction needs to happen physically at the practice. Follow-ups, prescription support, health reading reviews, documentation, and scheduled online consultations can often be supported digitally.

For patients, this can reduce unnecessary travel and make follow-up care easier to manage. For practitioners, it creates a more organised way to manage appointments, patient information, payments, telehealth sessions, and consultation history.

The platform is not positioned as a replacement for emergency care or physical consultations where examination is required. It provides a digital layer for appropriate private-practice healthcare interactions.

Problem

Healthcare admin can become scattered across tools.

Appointments, follow-ups, prescriptions, payments, sick notes, health readings, and patient records can become difficult to manage when they are handled manually or across separate systems.

Solution

A central platform for selected healthcare workflows.

TeleHealth gives patients and practitioners one web-based system for booking appointments, joining online consultations, handling consultation payments, and managing healthcare documents.

Development process

From account access to consultation flow.

Patient flow

From login to consultation.

  1. Register or login
  2. View dashboard
  3. Book appointment
  4. Pay consultation fee
  5. Join telehealth session
  6. Access documents and records
Practitioner flow

From schedule to care record.

  1. Login securely
  2. Manage appointment slots
  3. View patient details
  4. Conduct online consultation
  5. Create prescriptions or sick notes
  6. Review payments and history
System flow

From request to protected data.

  1. Validate user role
  2. Protect routes and sessions
  3. Store healthcare records
  4. Handle PayFast updates
  5. Support real-time calls
  6. Send notifications

Tech stack

HowTeleHealthwas built.

TeleHealth was built as a Flask and Supabase web application with server-rendered pages, database-backed healthcare workflows, online payments, and real-time telehealth support.

01 Frontend

HTML + CSS + JavaScript

Built the web interface, dashboard screens, forms, charts, and patient-facing flows.

02 Backend

Python Flask

Supported routes, sessions, authentication flow, business logic, and protected pages.

03 Database & Services

Supabase + PostgreSQL

Stored healthcare-related data and supported database-driven workflows.

04 Payments & Real Time

PayFast + SocketIO + WebRTC

Supported consultation payments, payment status handling, and online consultation rooms.

Python Flask Supabase PostgreSQL PayFast Socket.IO WebRTC Azure Communication Email
